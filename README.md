## 为什么数据采集用机房 IP 总被拦

**机房 IP 的根本问题不是技术问题，是身份问题。**

一个来自 AWS、阿里云、Linode 的 IP 地址，其 ASN 归属、网段注册信息、PTR 记录，全都指向数据中心。网站风控系统拿到这个 IP 的第一反应就是：这不是正常用户。接下来要么弹 CAPTCHA，要么直接 429/403，要么悄悄返回假数据。

所以，**做美国公开数据采集，IP 类型的选择比爬虫逻辑本身更重要**。换一个真正的美国住宅 IP，很多"技术问题"直接消失。

## 住宅 IP 的几种获取方式，各有什么坑

### 1. 共享住宅代理池

最常见的方案。Bright Data、Oxylabs 这类大厂提供按流量计费的住宅代理池，IP 来自真实设备，但一个 IP 可能被成百上千的用户轮流使用。IP 历史污染是大问题——前一个用户拿它做了什么你不知道，但目标网站的风控系统记得。

另外，代理池的出口节点在你和目标网站之间，中间多一层延迟，稳定性也依赖服务商的基础设施质量。

### 2. ISP 代理（静态住宅代理）

比动态代理池好一些，IP 固定、不轮换，但本质上还是代理出口，你的请求要经过服务商的节点再出去。在 IP 信誉上好于机房 IP，但比直接使用住宅宽带又差了一档。

### 3. 真实住宅宽带 VPS

这是最接近"真实美国用户上网"的方案。直接在美国本地公寓里拉一根家庭宽带，IP 由 AT&T 这类运营商分配给真实住宅地址，你在 VPS 上跑的任何请求，从目标网站视角来看，和洛杉矶某个普通家庭在浏览网页没有区别。

这种方案的问题是供应商稀少，且价格不低。但如果你的数据采集对 IP 纯净度有真实要求，这是性价比最高的方向。
